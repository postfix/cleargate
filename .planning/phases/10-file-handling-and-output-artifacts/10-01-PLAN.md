# Wave 1: Secure File Uploads

## Objective
Enforce size and extension limits dynamically while streaming file uploads, using limits defined in the tool's `ToolSpec`.

## Tasks

```xml
<task>
  <action>In `internal/api/upload.go`, add a `repo *repository.ToolSpecRepository` and `registry *job.Registry` dependency to `UploadHandler` and its constructor `NewUploadHandler`. Update `cmd/cleargate/main.go` to inject these when creating `UploadHandler`.</action>
  <read_first>
    - internal/api/upload.go
  </read_first>
  <acceptance_criteria>
    - `UploadHandler` has access to the tool spec for the job.
  </acceptance_criteria>
</task>

<task>
  <action>In `HandleUpload`, use the `jobID` to query the registry for the `toolID`. Then query the repo for the `ToolSpec`. Find the `ToolSpec.Inputs` that correspond to the form field name (`part.FormName()`). If the input isn't defined or doesn't match, reject the upload.</action>
  <read_first>
    - internal/api/upload.go
  </read_first>
  <acceptance_criteria>
    - Upload rejects files for undeclared inputs.
  </acceptance_criteria>
</task>

<task>
  <action>Enforce the `allowedExtensions` list from `ToolSpec.Inputs[x]`. Reject if `filepath.Ext(part.FileName())` is not in the list (case-insensitive).</action>
  <read_first>
    - internal/api/upload.go
  </read_first>
  <acceptance_criteria>
    - Upload rejects files with unapproved extensions.
  </acceptance_criteria>
</task>

<task>
  <action>Enforce `maxSizeMB` by wrapping the `part` reader in an `io.LimitReader`. For example, `limit := input.MaxSizeMB * 1024 * 1024`. Use `io.Copy(dst, io.LimitReader(part, int64(limit+1)))`. After copy, check if `limit > 0` and bytes written > limit. If exceeded, delete the partially written file and return a 400 error.</action>
  <read_first>
    - internal/api/upload.go
  </read_first>
  <acceptance_criteria>
    - Files exceeding `maxSizeMB` are blocked and truncated files are deleted.
  </acceptance_criteria>
</task>
```

## Verification
- File larger than limit returns error and leaves no residual file.
- Unauthorized extension returns error.
- Valid file uploads successfully.
