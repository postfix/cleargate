# Wave 2: Job Metadata & Output Retrieval

## Objective
Capture execution metadata post-run (exit code, output files, size) into `metadata.json` so the UI and APIs can retrieve the final job state and artifacts. Download logic is already partially functional.

## Tasks

```xml
<task>
  <action>Create a struct `JobMetadata` in `internal/models/toolspec.go` (or `internal/job`) containing: ExitCode, StdoutBytes, StderrBytes, OutputFiles (list of strings). Add fields to track bytes in `ExecutionHandler.HandleExecute`'s log stream loop.</action>
  <read_first>
    - internal/api/execute.go
  </read_first>
  <acceptance_criteria>
    - `JobMetadata` struct exists.
    - Bytes written to stdout/stderr are summed during stream.
  </acceptance_criteria>
</task>

<task>
  <action>In `ExecutionHandler.HandleExecute`, after `Wait()` completes, scan the `output/` directory using `os.ReadDir`. Collect the names of all generated files.</action>
  <read_first>
    - internal/api/execute.go
  </read_first>
  <acceptance_criteria>
    - Output directory is scanned.
  </acceptance_criteria>
</task>

<task>
  <action>Marshal the `JobMetadata` struct to JSON and write it to a `metadata.json` file in the job's workspace root (`h.workspaceManager.GetPath(req.JobID, "")`).</action>
  <read_first>
    - internal/api/execute.go
  </read_first>
  <acceptance_criteria>
    - `metadata.json` is correctly written with all fields populated post-execution.
  </acceptance_criteria>
</task>

<task>
  <action>Optionally, if an API is needed, add an endpoint `GET /api/jobs/{id}/metadata` that reads and serves `metadata.json`. Wire it up in `cmd/cleargate/main.go`. (Can just be served by a new handler or attached to execution handler).</action>
  <read_first>
    - internal/api/execute.go
  </read_first>
  <acceptance_criteria>
    - UI can request metadata from the backend.
  </acceptance_criteria>
</task>
```

## Verification
- `metadata.json` exists in the workspace after a job completes.
- It accurately reflects exit code and lists output files.
