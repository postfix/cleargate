# Wave 1: ToolSpec Validation & Versioning

## Objective
Implement struct validation for ToolSpec YAML schemas using `go-playground/validator/v10` and establish an append-only versioning strategy in the DuckDB repository.

## Tasks

```xml
<task>
  <action>Run `go get github.com/go-playground/validator/v10` to add the validation library. Add the following struct tags to `internal/models/toolspec.go`: 
- `Metadata.Name`: `validate:"required"`
- `Metadata.Version`: `validate:"required"`
- `Metadata.Runtime.TimeoutSeconds`: `validate:"required,gt=0"`
- `Flags[].ID`: `validate:"required"`
- `Flags[].Type`: `validate:"required"`
- `Inputs[].ID`: `validate:"required"`</action>
  <read_first>
    - internal/models/toolspec.go
  </read_first>
  <acceptance_criteria>
    - `go.mod` contains `github.com/go-playground/validator/v10`
    - `internal/models/toolspec.go` contains `validate:"required"` on all listed fields
  </acceptance_criteria>
</task>

<task>
  <action>Update `internal/api/admin.go` (specifically the drafts endpoint) and `internal/repository/toolspec_repo.go` (`SyncFromDirectory` method) to run `validator.New().Struct(spec)`. If validation fails, return an HTTP 400 response with a structured JSON error payload mapping field names to error messages.</action>
  <read_first>
    - internal/api/admin.go
    - internal/repository/toolspec_repo.go
  </read_first>
  <acceptance_criteria>
    - `internal/api/admin.go` imports `github.com/go-playground/validator/v10`
    - Draft POST endpoint returns HTTP 400 if `validator.Struct` returns an error
    - `SyncFromDirectory` logs a warning and skips files that fail validation
  </acceptance_criteria>
</task>

<task>
  <action>Update `ToolSpecRepository` in `internal/repository/toolspec_repo.go` to enforce append-only versioning (TOOL-03). In the `Approve` method (or related save methods), ensure that saving a new version of an existing ToolSpec uses an `INSERT` rather than `ON CONFLICT DO UPDATE`. Ensure the primary key in the DuckDB schema uses `PRIMARY KEY (id, version)`.</action>
  <read_first>
    - internal/repository/toolspec_repo.go
  </read_first>
  <acceptance_criteria>
    - `CREATE TABLE` query in `toolspec_repo.go` uses `PRIMARY KEY (id, version)`
    - `SaveDraft` or `Approve` methods do not use `ON CONFLICT` replacing existing versions
  </acceptance_criteria>
</task>
```
